Uber Performance Dashboard

This repository contains a comprehensive data visualization project analyzing Uber Auto service performance. The dashboard provides actionable insights into booking trends, revenue distribution, and operational efficiency across various vehicle types.

Project Overview

The dashboard serves as a strategic tool to monitor key business metrics, identifying growth patterns and service gaps. It features a clean, high-contrast UI designed for quick data interpretation by stakeholders.

Key Features

Performance Tracking: Real-time visibility into Completed Bookings (93K) versus Lost Bookings (57K).
Revenue Analysis: Breakdown of 52M total revenue, including a comparative analysis of vehicle types (Auto, Bike, Go Mini, etc.).
Operational Metrics: Monitoring of average trip distances (24.64 km) and total distance traveled (2.51M km).
Quality Control: Integrated rating system tracking both Customer (4.40) and Driver (4.23) feedback.
Geospatial Insights: Identification of high-traffic zones like Khandsa (Top Pickup) and Ashram (Top Drop).

Tech Stack

Data Visualization: Power BI Desktop

Data Processing: Power Query Editor, SQL, DAX, Data Modeling 


Insights Highlight

Booking Volatility: A notable dip in bookings during February suggests seasonal or external factors impacting demand.
Top Revenue Driver: The "Auto" category is the primary revenue engine, contributing 13M to the total.
Efficiency Ratios: Visual gauges for "Cancelled" and "Incomplete" rides provide immediate feedback on service reliability.
